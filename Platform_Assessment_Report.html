Critical Performance Bottlenecks

  • P1 N+1 query in priority_queue.py:118–119 — count_steps_for_contact() called per-row in a loop. 10 contacts = 11 queries. Fix: single JOIN query with window function.
  • P2 No connection pooling — get_connection() in database.py creates a new psycopg2 connection per request. Under load, this exhausts Supabase's 60-connection limit. Fix: psycopg2.pool.ThreadedConnectionPool.
  • P3 Migrations run per-request — run_migrations(conn) called in web/dependencies.py on every API call. That's 330 lines of SQL parsed per request. Fix: move to @app.on_event("startup").
  • P4 O(n²) fuzzy dedup — combinations(rows, 2) in deduplication.py. 10K companies = 50M comparisons. Fine for current 875 companies, but won't scale. Fix: cluster by first letter + length before comparing.
  • P5 Contact search uses LIKE without indexes — 5-column LIKE scan in contacts.py. Fix: add pg_trgm GIN indexes.
  • P6 No health check with DB ping — /api/health returns static JSON without checking DB connectivity.

Critical Security Issues

  • S1 No authentication — Zero auth middleware. All 50+ API endpoints publicly accessible on localhost. Anyone on the network can read/modify campaign data.
  • S2 Hardcoded CORS — Only allows localhost:5173. Will break in production. allow_methods=["*"] is overly permissive.
  • S3 Gmail token in project root — .gmail_token.json written to project directory. Risk of git commit. No encryption at rest.
  • S4 No file upload limits — CSV import has no size cap or MIME validation. DoS via large file upload.
  • S5 Jinja2 autoescape=False — Template engine doesn't escape HTML. XSS risk if user-controlled data reaches templates.
  • S6 PII in logs — Email addresses logged in plaintext via logger.info.
