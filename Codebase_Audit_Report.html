Critical Issues

Issues that cause performance degradation, data corruption, or resource exhaustion under normal usage.

Critical Performance Backend

N+1 Query Pattern in Queue Endpoint

src/web/routes/queue.py — lines 104-175

For each contact in the queue, the route runs separate queries for Gmail draft status and LinkedIn URL. With limit=25, that's 50+ extra DB round-trips per request. Dashboard becomes sluggish under normal use.

Fix: Batch-load all contact IDs upfront with WHERE contact_id IN (...) for both gmail_drafts and contacts tables.
Critical Performance Backend

N+1 Query Pattern in import_expandi.py

src/commands/import_expandi.py — lines 169-252

For each CSV row, individual SELECT queries find contacts and campaign status. A 1,000-row CSV causes 3,000+ round-trips instead of batch operations.

Fix: Pre-fetch all contacts by linkedin_url_normalized into a dict before the loop, then do O(1) lookups.
Critical Bug Backend

Unclosed Cursors — Connection Pool Exhaustion

Multiple files: reply_detector.py, whatsapp_scanner.py, template_engine.py, import_contacts.py, ab_testing.py, email_verifier.py

Cursors created via conn.cursor() but never explicitly closed. Under concurrent load, this pins connections in the pool until garbage collection runs, eventually exhausting available connections.

Fix: Add cursor.close() in finally blocks or use Python context managers throughout.
Critical Bug Backend

Race Condition in LinkedIn Acceptance Dedup

src/services/linkedin_acceptance_scanner.py — lines 246-255

Check-then-insert pattern using metadata LIKE %msg_id% is not atomic. Concurrent scans could process the same acceptance twice, advancing the sequence step twice.

Fix: Use a unique constraint or INSERT ... ON CONFLICT pattern for deduplication instead of check-then-insert.
Critical Performance Backend

N+1 Query in LinkedIn Acceptance Scanner

src/services/linkedin_acceptance_scanner.py — lines 199-214

For each of 50 matched messages, separate queries fetch enrollments and sequence steps. Results in 400+ unnecessary queries per scan.

Fix: Batch-load enrollments and cache sequence steps per campaign_id.
Critical Bug Frontend

Missing Null Safety on API Responses

frontend/src/pages/CampaignDetail.tsx — line 22

Destructures { campaign, metrics, variants, weekly, firm_breakdown } without null checks. If backend returns incomplete data, accessing metrics.by_status crashes the page.

Fix: Add optional chaining: metrics?.by_status || {} and default values for all destructured fields.
Critical Bug Frontend

Memory Leak in GlobalSearchBar

frontend/src/components/GlobalSearchBar.tsx — lines 14-43

Pending fetch requests complete on unmounted component, causing React state update warnings. No AbortController used.

Fix: Add AbortController to the fetch and abort in the cleanup function.

High-Severity Issues

Issues that affect reliability, data quality, or user experience under edge conditions.

HighBugBackend

Missing Transaction Boundaries in CSV Import

src/commands/import_contacts.py — lines 171-209

Entire CSV import runs in a single transaction. If row 500 of 875 fails, all 499 previous inserts roll back with no resume capability.

Fix: Use savepoints per company/contact batch, track failed rows, allow resume from last success.
HighBugBackend

Silent Failures in External API Calls

src/services/email_verifier.py — lines 110-145

ZeroBounce/Hunter API errors silently mark all emails as "unknown" without logging details. No API key validation at startup. Configuration issues go undetected.

Fix: Add structured logging, validate API keys on first use, implement retry with backoff.
HighBugBackend

No Transaction Isolation for State Machine

src/services/state_machine.py, linkedin_acceptance_scanner.py

Concurrent campaign operations could interleave contact status updates. PostgreSQL defaults to READ COMMITTED, allowing race conditions in step advancement.

Fix: Use SELECT ... FOR UPDATE or advisory locks on contact_id during state transitions.
HighDatabase

Missing Performance Indexes

migrations/pg/

Several frequently-queried columns lack indexes: pending_replies.confirmed, response_notes.note_type, whatsapp_messages.direction, templates(is_active, channel) composite, dedup_log columns.

Fix: Add a new migration (008) with targeted indexes on these columns.
HighQualityFrontend

40+ any Types in API Client

frontend/src/api/client.ts — throughout

Most API methods return request<any>(), defeating TypeScript. Refactoring is risky and bugs are hidden at compile time.

Fix: Define response types for all endpoints and replace any with proper generics.
HighBugFrontend

Missing Error UI for Pipeline Drag-and-Drop

frontend/src/pages/Pipeline.tsx — lines 406-442

Stage mutation has rollback logic but never shows users that an error occurred. Deals silently snap back without explanation.

Fix: Add error toast or inline error when stageMutation.isError.
HighBugFrontend

Stale Closures in QueueEmailCard

frontend/src/components/QueueEmailCard.tsx — lines 91-105

handleSaveEdit() captures form values via closure. If state updates while promises are pending, mutations use stale values.

Fix: Capture values at mutation time or use refs.
HighQualityFrontend

No Focus-Visible Styles (Accessibility)

All components

Buttons and inputs lack focus-visible:ring-2 outlines. Keyboard navigation has no visual indicator — WCAG 2.1 AA violation.

Fix: Add global focus-visible:ring-2 ring-blue-500 ring-offset-2 to index.css.
HighQualityFrontend

No ARIA Landmarks

frontend/src/components/Layout.tsx

Missing <nav>, <main>, <aside> semantic elements. Screen readers can't navigate the app structure.

Fix: Wrap sidebar in <aside>, content in <main>, nav links in <nav>.
HighQualityBackend

Inconsistent Error Response Format

src/services/llm_advisor.py, reply_detector.py

On error, some services return JSON strings while others return dicts. Callers can crash on type mismatches.

Fix: Always return structured dicts with an error key.

Architecture Notes

Note

Connection Pool Lifecycle

Pool initializes at startup but routes lack middleware for cleanup on exception. Add an HTTP middleware that ensures connections are returned to the pool even on unhandled errors.

Note

Template Selector Cold-Start Bias

The explore/exploit algorithm in template_selector.py locks onto early winners too fast. Untested templates get score 0.0 in exploit mode. Consider guaranteeing each template gets a minimum 5 sends before locking in.

Note

Duplicate Import Logic

import_contacts.py, import_emails.py, and import_expandi.py all follow CSV-read → normalize → insert patterns. Could be refactored into a shared csv_import.py service.
