Top Simplification Opportunities

1. Extract QueryBuilder helper — saves ~400 lines across 15 routes

Every list endpoint repeats WHERE 1=1 + conditional AND + params.append(). A helper like QueryBuilder().where("status = %s", val).order_by("id").limit(n).build() cuts each list function from ~25 lines to ~8.

2. Centralize frontend API + constants — saves ~350 lines

Create src/constants.ts (lifecycle stages, status types, colors, query keys) and upgrade api/client.ts with typed params + error response types. Eliminates repeated definitions in 4+ page files.

3. Split 4 god components — makes each 150-200 lines

ResearchJobDetail → CompletionSummary, BatchImportModal, ResultsGrid. ContactDetail → ContactTabs, ContactTimeline, ContactForm. Pipeline → StageColumn, DealCard, PipelineFilters. Each extraction is a standalone PR.

4. Split crypto_research.py into 4 modules — 1,078 → ~270 each

crypto_api.py (API clients), crypto_scoring.py (scoring), crypto_cache.py (caching), crypto_report.py (formatting/LLM). Single responsibility per module.

5. Python Enums + TS union types for statuses — eliminates magic strings

Define class ContactStatus(str, Enum) and class Channel(str, Enum) in src/constants.py. Frontend gets a matching type ContactStatus = "queued" | "in_progress" | .... Typos become errors, not silent bugs.

6. Standardize error responses with FastAPI exception handler

Register a custom AppError exception + handler. Routes raise AppError(400, "msg"), middleware formats. Removes ~100 lines of manual JSONResponse(content={"error": ...}).
