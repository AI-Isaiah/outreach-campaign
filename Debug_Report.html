1. Backend Bugs

10 issues found across Python services and FastAPI routes. The dominant pattern is unclosed database cursors in web route handlers.

CRITICAL B001
30+ Unclosed Database Cursors Across All Web Routes
src/web/routes/*.py — all 13 route files
Every route handler creates cursors with conn.cursor() but never calls cursor.close(). While the service layer (state_machine, compliance, priority_queue) has proper try/finally cleanup, the web layer does not.
Under sustained load, unclosed cursors accumulate server-side resources. Can exhaust the connection pool and crash the app.
Fix: Wrap all cursor operations in try/finally blocks: cur = conn.cursor(); try: ... finally: cur.close(). Or create a shared cursor context manager in dependencies.py.
HIGH B002
Hardcoded localhost Redirect in Gmail OAuth Callback
src/web/routes/gmail.py:62 & src/services/gmail_drafter.py:81,104
OAuth callback redirects to http://localhost:5173/settings?gmail=connected and the OAuth redirect URI is hardcoded to http://localhost:8000/api/gmail/callback. Both break in production.
Gmail integration will not work in any deployed environment (Railway, Docker, etc.).
Fix: Use FRONTEND_URL and BACKEND_URL environment variables. Default to localhost for dev.
MEDIUM B003
Race Condition: Lock Released Before Enrollment
src/services/state_machine.py:115-145
Uses FOR UPDATE OF c to lock the next contact, but enrollment happens after cursor.close() releases the lock. Two concurrent transitions could activate the same contact.
Violates "one contact per company" rule under concurrent usage.
Fix: Move enroll_contact() inside the try block before cursor.close().
MEDIUM B004
Conditional conn.commit() in update_contact_status
src/web/routes/contacts.py:170-232
The commit only happens if body.note is provided. State transitions and event logging may not be committed without a note.
Status changes could be lost if the user doesn't provide a note.
Fix: Always call conn.commit() at end of function, regardless of note.
MEDIUM B005
N Unclosed Cursors in Queue Enrichment Loop
src/web/routes/queue.py:99-141
Inside the queue enrichment loop, new cursors are created per item without closing. For a queue of 20 items, that's 40+ leaked cursors per request.
Significant resource leak proportional to queue size.
Fix: Reuse a single cursor for the entire loop, close in finally block.
MEDIUM B006
WhatsApp Scan Status — 3 Queries, No Cursor Close
src/web/routes/whatsapp.py:58-84
Three separate database queries without cursor cleanup.
Resource leak.
Fix: try/finally around all three queries.
MEDIUM B007
Insights History — Unclosed Cursor + Inconsistent Branching
src/web/routes/insights.py:44-57
When no campaign_id, creates cursor without closing. When campaign_id present, delegates to service (which may also leak).
Resource leak.
Fix: Add try/finally.
LOW B008
Missing Exception Handling in Deals Endpoints
src/web/routes/deals.py
Database failures propagate as unhandled 500 errors without logging.
Fix: Add try/except with logging.
LOW B009
Potential Open Redirect Risk
src/web/routes/gmail.py:62
Currently hardcoded (safe), but if changed to dynamic redirect without validation, could be exploited.
Fix: When making B002 fix, use URL allowlist validation.
LOW B010
9+ Unclosed Cursors in models/campaigns.py
src/models/campaigns.py
Model-layer CRUD functions create cursors without try/finally.
Fix: Add try/finally to all model functions.

2. Frontend Bugs

23 issues found across React pages, components, and the API client. Dominant patterns: missing error states, type safety gaps, and direct state mutation.

HIGH F001
7 Pages Missing Error State Display
Queue.tsx, Insights.tsx, Pipeline.tsx, Dashboard.tsx, ContactDetail.tsx (deals query), CampaignDetail.tsx, Templates.tsx
useQuery hooks fetch data but don't render error messages when queries fail. Users see blank sections or infinite loading spinners.
Fix: Add {query.isError && <div className="text-red-500">Failed to load</div>} pattern to all queries.
HIGH F002
Direct State Mutation in ContactDetail.tsx
ContactDetail.tsx:75-76
statusForm.campaign = enrollments[0].campaign_name directly mutates state outside setState, bypassing React's reactivity system.
Component may not re-render when default campaign is set. Stale form state.
Fix: Use useEffect with setStatusForm to set default campaign.
HIGH F003
Stale Query Invalidation on Paginated Timeline
ContactDetail.tsx:39 vs UnifiedTimeline.tsx:50
Note mutation invalidates ["timeline", contactId] but the actual query key is ["timeline", contactId, page]. Pages 2+ won't refresh after adding a note.
Fix: Use queryClient.invalidateQueries({ queryKey: ["timeline", contactId], exact: false }).
HIGH F004
Non-Null Assertions on Route Params
CampaignDetail.tsx:11, Templates.tsx:48
Using name! and editingId! without runtime checks. If params are undefined, API receives undefined.
Fix: Add guard before mutation: if (!name) throw new Error('Missing param').
HIGH F005
Dependent Query Race Condition
ContactDetail.tsx:29-33
Deals query depends on data?.contact?.company_id from another query. Could use stale data if the parent query re-fetches.
Fix: Add && !isLoading to the enabled condition.
HIGH F006
Persisting Mutation Data Shows Stale Results
Dashboard.tsx:78-82
After scanning replies, scanReplies.data persists across page navigations, showing old scan results as if they were current.
Fix: Reset mutation on unmount: useEffect(() => () => scanReplies.reset(), []).
MEDIUM F007
"null null" Rendered as Contact Name
CompanyDetail.tsx:110
{c.full_name || `${c.first_name} ${c.last_name}`} renders "null null" when both names are null.
Fix: {c.full_name || `${c.first_name || ''} ${c.last_name || ''}`.trim() || '-'}
MEDIUM F008
Index Used in List Keys
Inbox.tsx:66, UnifiedTimeline.tsx:61
Keys include array index which changes on reorder/pagination, causing React reconciliation issues.
Fix: Use only stable identifiers: key={`${item.channel}-${item.item_id}`}.
MEDIUM F009
GlobalSearchBar Memory Leak on Unmount
GlobalSearchBar.tsx:14-32
Cleanup clears timeout but doesn't cancel in-flight API requests. setResults() may fire on unmounted component.
Fix: Use AbortController or a cancelled flag.
MEDIUM F010
Missing Channel-Specific Template Validation
Templates.tsx:189
Email templates require a subject line but the form only validates name and body_template.
Fix: Add || (form.channel === 'email' && !form.subject) to disabled check.
MEDIUM F011
Pipeline Deal Creation — No company_id Validation
Pipeline.tsx:366
Number("") returns 0, which creates deals with company_id=0 (invalid).
Fix: Validate before submission: if (!companyId) return.
MEDIUM F012
Unsafe Array Access on Dashboard
Dashboard.tsx:84-86
pendingReplies.data.slice(0, 5) may throw if data becomes undefined mid-render.
Fix: Use optional chaining: pendingReplies.data?.slice(0, 5).
MEDIUM F013
23 Instances of any Type in API Client
api/client.ts
Almost every endpoint returns request<any>, defeating TypeScript's type safety.
Fix: Create response interfaces for each endpoint.
MEDIUM F014-F017
Missing Loading/Error States on Pagination (4 pages)
Inbox.tsx, UnifiedTimeline.tsx, ContactList.tsx, Companies listing
Pagination buttons don't disable during loading, allowing rapid clicks that queue requests.
Fix: Add disabled={isLoading} to pagination buttons.
LOW F018
Native confirm() Dialog on Pipeline Delete
Pipeline.tsx:267
Uses browser confirm() instead of a custom modal. Looks unprofessional.
Fix: Replace with a confirmation modal component.
LOW F019
Missing ARIA Labels on Form Inputs
GlobalSearchBar, Templates, Pipeline forms
Search bars and inputs lack aria-label attributes.
Fix: Add aria-label to all inputs.
LOW F020
Error Boundary Has Minimal Recovery UX
App.tsx:30-36
ErrorBoundary shows "Try again" but no error details or logging. Users can't report what went wrong.
Fix: Show error summary, add "Copy error" button, log to console.

3. Database & Schema Issues

18 issues found in migrations, schema design, and connection handling.

CRITICAL D001
Polymorphic entity_tags Has No Foreign Key on entity_id
migrations/pg/006_crm_pipeline.sql:38-44
The entity_tags table references contacts or companies via entity_type + entity_id but has no FK constraint. If a contact is deleted, orphaned tag references remain.
Fix: Either split into contact_tags/company_tags with proper FKs, or add a cleanup trigger.
CRITICAL D002
dedup_log Has No Foreign Keys
migrations/pg/001_initial_schema.sql:95-102
kept_contact_id and merged_contact_id reference contacts but have no FK constraints. Audit trail breaks when contacts are deleted.
Fix: Add FK constraints with ON DELETE SET NULL (audit trail should survive contact deletion).
CRITICAL D003
9+ Unclosed Cursors in models/campaigns.py
src/models/campaigns.py
Model-layer functions (update_contact_campaign_status, log_event, etc.) create cursors without try/finally.
Fix: Add try/finally to all model functions.
HIGH D004
Missing UNIQUE on email_normalized
migrations/pg/001_initial_schema.sql:105
Index exists but no UNIQUE constraint. Code assumes uniqueness for deduplication but the DB doesn't enforce it.
Fix: CREATE UNIQUE INDEX idx_contacts_email_norm ON contacts(email_normalized) WHERE email_normalized IS NOT NULL
HIGH D005
Missing UNIQUE on linkedin_url_normalized
Same as D004
Same issue for LinkedIn URLs. Two contacts with same LinkedIn can exist.
Fix: Add UNIQUE partial index.
HIGH D006
Unbounded TEXT Columns — No Length Limits
migrations/pg/001_initial_schema.sql
All string columns use TEXT type. Emails, URLs, names have no length validation at the DB layer.
Fix: Use VARCHAR(255) for emails, VARCHAR(500) for URLs, VARCHAR(100) for names.
HIGH D007
Missing Indexes on Timeline View Backing Tables
interaction_timeline_view + events, gmail_drafts, pending_replies, response_notes
The view does UNION ALL across 5 tables with WHERE contact_id = ? and ORDER BY occurred_at DESC but backing tables lack composite indexes on (contact_id, timestamp).
Fix: Add composite indexes: CREATE INDEX idx_events_contact_ts ON events(contact_id, occurred_at DESC) etc.
MEDIUM D008
Missing Composite Index on contact_template_history
migrations/pg/005_performance_indexes.sql
Insights queries by campaign + outcome but only has single-column index on campaign_id.
Fix: Add CREATE INDEX idx_cth_campaign_outcome ON contact_template_history(campaign_id, outcome)
MEDIUM D009
No Transaction Isolation or Statement Timeout
src/models/database.py:11-22
Connection pool has no statement timeout configured. A runaway query could hold a connection indefinitely.
Fix: Add options='-c statement_timeout=30000' to connection string.
MEDIUM D010
No Retry Logic in get_pool_connection()
src/web/dependencies.py:29-35
If pool is exhausted, getconn() raises immediately. No retry with backoff.
Fix: Add retry with exponential backoff (3 attempts, 100ms base).
MEDIUM D011-D014
Missing NOT NULL and CHECK Constraints (4 issues)
001_initial_schema.sql
contacts.email allows NULL (should it?), email_status has no CHECK constraint against valid values, deals FK missing cascade config.
Fix: Add constraints in a new migration.
LOW D015-D018
Minor Index & Type Issues (4 issues)
Various migrations
LIKE vs ILIKE inconsistency, unused enum values, missing soft-delete audit trail.
Fix: Quality-of-life improvements in future migration.
